# Lịch sử Phiên Làm Việc - Ngày 17/07/2026

## Tóm tắt công việc
- **Tác vụ**: Sửa lỗi 500 (Internal Server Error) và đồng bộ API cho Module Sinh viên để phục vụ tích hợp Android App.
- **Phân hệ**: `sdms-backend`

## Chi tiết các thay đổi
1. **Sửa lỗi 500 (NullPointerException & 404 Exception)**
   - Cập nhật `CurfewRequestService` (`getMyRequests`): Thêm kiểm tra null cho `account.getStudent()`, trả về `Page.empty()` thay vì ném lỗi, đảm bảo dữ liệu `[]` được trả về khi sinh viên chưa có hồ sơ.
   - Cập nhật `StudentService` (`getMyProfile`): Trả về `null` thay vì ném `AppException(RESOURCE_NOT_FOUND)` khi không tìm thấy hồ sơ sinh viên, giúp controller trả về `data: null` một cách hợp lệ, không gây 500 error cho mobile.
   - Cập nhật `StudentChangeRoomController`: Bổ sung kiểm tra null cho `account.getStudent()` ở các endpoints `submitRequest` và `getMyRequests` để tránh sập hệ thống (NPE) khi tài khoản rỗng.

2. **Cung cấp API "Danh sách phòng trống"**
   - **File tạo mới**: `StudentRoomController.java` (đặt tại `com.sdms.backend.modules.student.controller`).
   - Cung cấp API `GET /api/v1/student/rooms/available` dành riêng cho ROLE_STUDENT.
   - Tự động lọc các phòng đang có `status == AVAILABLE` và `occupiedBeds < capacity`.
   - Trả về cấu trúc: `roomId`, `roomCode`, `buildingName`, `floorNumber`.

3. **Chuẩn hóa cấu trúc Response (Sync with BaseResponse)**
   - Bỏ thuộc tính `@JsonInclude(JsonInclude.Include.NON_NULL)` khỏi `ApiResponse.java`.
   - Lỗi trước đó: Jackson tự động lược bỏ các trường bị `null` (như `data`, `errorCode`) khiến thư viện Gson/Moshi trên Android báo lỗi thiếu key khi parse. 
   - Hiện tại: Tất cả các trường `success`, `message`, `data`, `errorCode` luôn được giữ lại trong JSON Response, giải quyết triệt để lỗi Parse JSON thô.

4. **Kiểm tra Logic Đổi phòng (ID Spoofing)**
   - Hàm `submitRequest` trong `ChangeRoomService.java` hiện đã truy xuất `assignmentRepository.findByStudent_StudentIdAndStatus(studentId, AssignmentStatus.OCCUPIED)`.
   - Tính năng chặn ID Spoofing đã được đảm bảo an toàn vì hệ thống chỉ lấy `studentId` từ SecurityContextHolder (JWT của phiên đăng nhập hiện tại), Android app không cần (và không thể) truyền lên thông tin phòng cũ của sinh viên khác.

## Đã kiểm thử & xác nhận
- Toàn bộ backend đã pass qua bước `mvn compile` thành công. Không có lỗi biên dịch.
