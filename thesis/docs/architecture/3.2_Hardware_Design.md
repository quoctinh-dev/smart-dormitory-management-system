## 3.2.1 Sơ đồ kiến trúc phần cứng tổng thể

Hệ thống phần cứng IoT của dự án SDMS được chia thành hai cụm thiết bị độc lập, đảm nhiệm hai không gian vật lý khác nhau: Cổng chính Ký túc xá và Cửa phòng lưu trú. Cả hai cụm thiết bị đều sử dụng họ vi điều khiển ESP32 nhờ ưu điểm tích hợp sẵn kết nối Wi-Fi, chi phí hợp lý và tài nguyên phần cứng (RAM/Flash) đủ lớn để xử lý dữ liệu cảm biến cũng như kết nối HTTP/MQTT bảo mật.

## 3.2.2 Cụm thiết bị Cổng Ký túc xá (Smart Access Gate)

Cụm cổng Ký túc xá sử dụng bo mạch **AI-Thinker ESP32-CAM** làm trung tâm xử lý, tích hợp bộ nhớ PSRAM 4MB để có khả năng bắt hình ảnh độ phân giải cao phục vụ mô hình AI nhận diện khuôn mặt. Thiết bị này phụ trách việc thu thập thông tin xác thực đa phương thức (Khuôn mặt và Thẻ RFID) đồng thời điều khiển cơ cấu khóa cổng.

### 1. Thành phần linh kiện
- **Vi điều khiển:** ESP32-CAM (Tích hợp module Camera OV2640).
- **Mô-đun đọc thẻ:** MFRC522 (Giao tiếp qua chuẩn SPI).
- **Cơ cấu chấp hành (Actuator):** Động cơ Servo 360 độ (Đóng/mở chốt khóa vật lý).

### 2. Sơ đồ đấu nối (Pin Mapping)
Sơ đồ chân tín hiệu đã được tối ưu hóa để tránh xung đột với các chân Strapping Pins của ESP32 (đặc biệt là GPIO 0, 2, 12, 15) trong quá trình khởi động (Boot mode). Riêng chân RST của RC522 được đấu trực tiếp vào nguồn 3.3V thay vì dùng GPIO4 nhằm tránh làm sáng đèn LED Flash công suất cao tích hợp trên bo mạch ESP32-CAM.

| Linh kiện | Chân tín hiệu linh kiện | Chân GPIO trên ESP32-CAM | Ghi chú |
|-----------|--------------------------|--------------------------|---------|
| **RC522 (RFID)** | SDA / SS | GPIO 14 | Chuẩn giao tiếp HSPI |
| | SCK | GPIO 15 | |
| | MOSI | GPIO 2 | Cấu hình kéo mức thấp (LOW) lúc khởi động |
| | MISO | GPIO 13 | |
| | RST | 3.3V | Bỏ qua GPIO để tránh kích hoạt Flash LED |
| **Servo Motor** | PWM Signal | GPIO 12 | Tín hiệu PWM điều khiển góc quay |
| **Camera OV2640**| (Cáp Ribbon) | DVP Bus (Internal) | Kết nối cố định trên bo mạch |

## 3.2.3 Cụm thiết bị Cửa phòng (Room Door Controller)

Cụm thiết bị cửa phòng lưu trú sử dụng bo mạch **ESP32 DevKit V1 (30-pin)** làm trung tâm điều khiển. Khác với cụm cổng chính yêu cầu camera, cửa phòng tập trung vào tương tác người dùng tại chỗ (giao diện HMI cơ bản) thông qua mã PIN bảo mật. Thiết bị có khả năng hoạt động ở cả hai chế độ: Trực tuyến (Xác thực qua Backend) và Ngoại tuyến (Offline mode — mở cửa bằng Master PIN dự phòng).

### 1. Thành phần linh kiện
- **Vi điều khiển:** ESP32 DevKit V1.
- **Giao diện nhập liệu:** Bàn phím ma trận cứng 4x4 (Keypad).
- **Giao diện hiển thị:** Màn hình LCD 1602 kết hợp module I2C.
- **Cơ cấu chấp hành:** Động cơ Servo (Mô phỏng khóa chốt cửa).

### 2. Sơ đồ đấu nối (Pin Mapping)
Các linh kiện được phân bổ GPIO để bảo đảm không xảy ra xung đột tín hiệu, đặc biệt là tách biệt bus I2C (cho màn hình) và các chân Digital Input (cho bàn phím).

| Linh kiện | Chân tín hiệu linh kiện | Chân GPIO trên ESP32 | Ghi chú |
|-----------|--------------------------|----------------------|---------|
| **LCD 1602 (I2C)** | SDA | GPIO 21 | Giao tiếp I2C mặc định |
| | SCL | GPIO 22 | |
| **Keypad 4x4** | ROW 1 | GPIO 14 | Quét hàng ma trận |
| | ROW 2 | GPIO 27 | |
| | ROW 3 | GPIO 26 | |
| | ROW 4 | GPIO 25 | |
| | COL 1 | GPIO 33 | Đọc cột ma trận |
| | COL 2 | GPIO 32 | |
| | COL 3 | GPIO 4 | |
| | COL 4 | GPIO 16 | |
| **Servo Motor** | PWM Signal | GPIO 13 | Điều khiển chốt khóa phòng |

## 3.2.4 Yêu cầu về môi trường và kết nối

Để bảo đảm tính sẵn sàng cao, các cụm phần cứng IoT được thiết kế với các thông số kết nối tiêu chuẩn sau:
- **Giao thức mạng:** Giao tiếp qua mạng nội bộ Wi-Fi (chuẩn 802.11 b/g/n băng tần 2.4GHz).
- **Cơ chế phục hồi:** Tích hợp bộ định tuyến khởi động lại vòng lặp (Watchdog/Reconnect loop) sau mỗi 5 giây (`RECONNECT_INTERVAL = 5000`) nếu phát hiện rớt kết nối Wi-Fi hoặc mất liên lạc với MQTT Broker.
- **Thời gian khóa tự động:** Cả hai hệ thống đều thiết lập thời gian ngắt rơ-le (chốt khóa tự đóng lại) mặc định là 5 giây (`RELAY_OPEN_DURATION = 5000`) sau khi xác thực thành công.

## GHI CHÚ KỸ THUẬT (Cho ChatGPT khi soạn văn bản)
- Phần này thể hiện chuyên môn nhúng (Embedded) rõ rệt. Nhấn mạnh vào việc xử lý rủi ro phần cứng như "Tránh xung đột chân Strapping của ESP32" và "Chống lỗi Boot loop" — đây là yếu tố ăn điểm về mặt học thuật.
- Bảng chân GPIO phải giữ nguyên, không được tự ý sửa số pin vì nó khớp chính xác với mã nguồn C++ của thiết bị thật.
