# Session Work Log — 2026-07-20

## Objective
Fix HTTP 500 error on `GET /api/v1/bills/me/paged` called by the Mobile App.

## Root Cause Analysis

### Primary Cause: Missing Endpoint
The Mobile App called `GET /api/v1/bills/me/paged` but no such mapping existed in `BillController`. Spring MVC could not route the request, which caused the `INTERNAL_SERVER_ERROR` response.

### Secondary Causes (Architecture Violations)
The original `BillController` had several violations of the Backend Architecture Contract (AGENTS.md):
1. **Illegal Repository Injection**: Controller directly injected `BillRepository`, `DormitoryApplicationRepository`, `StudentRepository`, and `RoomRepository` — bypassing the Service layer.
2. **Raw Entity Exposure**: `/me` endpoint returned `List<Bill>` (JPA entity) directly, violating the Entity → Mapper → DTO → ApiResponse rule.
3. **Missing Service Delegation**: All business logic (fetching applications by CCCD, enriching names) was done in the Controller instead of the Service.

## Files Modified

### 1. `BillController.java` — Full Refactor
- **Removed** all direct repository injections (4 repos removed).
- **Added** `private final BillService billService` — single dependency.
- **Refactored** `/me` endpoint to delegate to `billService.getMyBills()`.
- **Added** new `GET /me/paged` endpoint delegating to `billService.getMyBillsPaged()`.
- **Refactored** `GET /` (admin) to delegate to `billService.getAllBillsPaged()`.
- **Changed** `/application/{applicationId}` to return `BillResponse` DTO.

### 2. `BillService.java` — New Methods Added
- **Added** `getBillByApplicationId(UUID)` — returns `BillResponse` DTO.
- **Added** `getMyBills(UserAccount)` — non-paged list, merges accommodation + utility bills.
- **Added** `getMyBillsPaged(UserAccount, Pageable)` — paginated version for Mobile App.
- **Added** `getAllBillsPaged(Pageable)` — admin-facing paginated view with student name enrichment.
- **Added** `private toBillResponse(Bill)` — reusable Entity → DTO mapper.
- **Added** fields: `StudentRepository`, `RoomRepository`, `DormitoryApplicationRepository`.

### 3. `BillRepository.java` — New Query Methods
- **Added** `findByStudentId(UUID)` — fetch all bills by studentId.
- **Added** `findByApplicationIdIn(List<UUID>, Pageable)` — paginated by applicationIds.
- **Added** `findByStudentIdIn(List<UUID>, Pageable)` — paginated by studentIds.
- **Added** `findAllMyBills(List<UUID>, UUID, Pageable)` — merged paginated query (applicationIds OR studentId).
